# DESIGN

## 設計原則

- 各 Scene、Node には、可能な範囲で焦点の定まった責務を持たせる。
- 機能を組み合わせる場合はコンポジションを優先する。
- 継承は、明確な型の関係と共通の契約がある場合に使用する。
- 外部依存を明示し、必要な箇所では依存先を差し替えられる構造にする。

## 設計上の注意

- Node 数やコード行数だけを理由に Scene やクラスを分割しない。
- 将来必要になる可能性だけを理由に、抽象化やグローバルな仕組みを追加しない。

## 所有関係と依存関係

- 複数の Node の関係を把握している所有者が、依存関係の設定や Signal の接続を担当する。
- 外部依存は、`@export`、公開プロパティ、初期化メソッド、Callable、Resource などを使用して明示的に提供する。
- 外部依存を取得するために SceneTree 全体を探索する実装は避ける。

## オブジェクト間の連携

- 何かが発生したことを通知する場合は Signal を使用する。
- 特定の処理を要求する場合はメソッドを使用する。
- 処理そのものを外部から提供する場合は Callable を使用する。

## Godotの構成要素

### Scene

- 再利用可能な Scene は、可能な範囲で自己完結させる。
- Scene 内部の Node 構造は実装詳細として扱う。
- Scene 外部からの操作は、Scene Root または明確な公開APIを通して行う。
- SceneTree は、表示上の階層だけでなく、所有関係とライフサイクルを考慮して構成する。
